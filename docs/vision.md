# PaletteAI -- Vision

## North Star

PaletteAI turns a plain-language brand description into a production-ready design token system in under 30 seconds. No design expertise required. The output is immediately usable in Figma, CSS, Tailwind, or any token-consuming tool.

## Target Audience

1. **Solo founders and indie hackers** who need a cohesive visual identity but lack a designer.
2. **Frontend developers** who want a consistent token system without hand-picking every value.
3. **Design system teams** looking for rapid exploration and starting points for new brands or sub-brands.
4. **Agencies** pitching multiple brand concepts quickly during discovery phases.

## Design Principles

### 1. Instant gratification
The user should see a beautiful, complete result within seconds of describing their brand. No multi-step wizards, no account creation walls.

### 2. Opinionated defaults, full control
AI generates a complete system with strong defaults. Every token is individually adjustable after generation. The tool has an opinion so the user does not have to.

### 3. Production-ready output
Generated tokens are not mockups. They are real, well-structured CSS custom properties, JSON tokens, or Figma variables ready to drop into a project.

### 4. Accessibility baked in
Every generated palette is checked against WCAG 2.1 contrast ratios. The AI is instructed to produce accessible combinations by default. Violations are flagged, not silently shipped.

### 5. Show, don't tell
Every token is previewed in context: color swatches with text overlays, typography samples in realistic layouts, spacing demonstrated on actual UI components.

## AI Durability

The AI layer is a generation engine, not a runtime dependency. Once tokens are generated, the entire output is static and self-contained. If the AI service is unavailable, previously generated systems remain fully functional. The app gracefully degrades to manual token editing when AI is offline.
